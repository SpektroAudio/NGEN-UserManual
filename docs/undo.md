# Undo

---

## About Undo System
The Undo System in NGEN lets users quickly revert to previously generated sequences for Generators / MIDI FXs / Clocks by pressing ++"GENERATE"++ + ++"RETURN"++.

When a new sequence is generated by a Generator / MIDI FX / Clock, NGEN caches the current state of that part, including the parameter values, before the new generation.
In order to keep memory usage reasonable, NGEN caches the last 16 generated sequences in memory.

The Undo System is currently used for:

| Generators | MIDI FXs  | Clock Generators |
|------------|-----------|------------------|
| ACDGEN     | Accent    | FreeClock        |
| Arper      | Modulator |                  |
| DrumGen    | ProgSeq   |                  |
| Luar       |           |                  |
| MARP       |           |                  |
| NSL Engine |           |                  |
| Polyform   |           |                  |
| Pop        |           |                  |
| Samba      |           |                  |
| Shuffler   |           |                  |
| Turing     |           |                  |


## Limitations

- Sequences are only saved to the undo cache when a new sequence is generated.
N
- Parameter changes during pattern generations are not saved / cached.

- When triggering a [Pattern Generation](pattern.md#pattern-generation), NGEN will store the content of each track generator and midi fx as individual sequences. This means that while the previous sequences are stored to the undo cache (and can be recall individually), it's not possible to revert a Pattern Generation as a whole.

- Generators (like Thru) and MIDI FXs (like Chords or Glitch) that don't use an internal sequence are not cached.

- Cached sequences can only be reverted to the same Track and Generator / MIDI FX.

- The undo cache is not persistent between reboots and it's not saved to the projects.
